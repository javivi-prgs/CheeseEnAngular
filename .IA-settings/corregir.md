Fase 1: 
    Cimentación y Diseño (Semana 1)El objetivo es pasar de la idea al código base.
    Definición de Contenidos: Investigar y agrupar toda la información, imágenes y videos que llevará la web.
    Identidad Visual: Seleccionar la paleta de colores y tipografías. Maquetar cómo se verá el sitio.
    Setup Técnico: 
    * Crear el proyecto en Angular.
    Configurar el repositorio en GitHub para el trabajo colaborativo.
    Instalar e integrar Bootstrap para asegurar que el diseño sea responsivo desde el inicio.
    
🏗️ Fase 2: Arquitectura y Navegación (Semana 2)Aquí es donde la web empieza a "moverse".
    Estructura de Componentes: Crear los componentes básicos (Navbar, Footer, Home, etc.).
    Enrutamiento Dinámico: * Configurar el RouterOutlet en el componente principal.
    Implementar routerLink en el Navbar para navegar sin recargar la página.
    Maquetación de Inicio: * Diseñar un Navbar responsivo con links a Registro/Login.
    Añadir secciones de Cover (portada), Carrusel y el apartado de "Quiénes Somos".
    Configurar el Footer con iconos de redes sociales.

📊 Fase 3: Datos y Servicios (Semana 3)Es el momento de conectar la lógica de programación.
    Gestión de Datos (JSON): Elaborar un archivo JSON que contenga la información de tus productos, servicios o usuarios.
    Servicios de Angular: Crear un servicio para realizar peticiones (HttpClient) y consumir los datos del JSON o una API externa.
    Catálogo Dinámico: * Crear el componente de Catálogo que muestre los productos mediante un bucle (*ngFor).
    Implementar botones de "Ver Detalles" que lleven a la información específica de cada artículo.
    Diseñar la vista de Detalle de Producto usando dos columnas con Bootstrap.

📝 Fase 4: Formularios y Cierre (Semana 4)
    Interactividad final y control de calidad.
    Formularios con Validación: * Crear componentes de Registro e Inicio de Sesión.
    Implementar el Formulario de Contacto.
    Importante: Todos los inputs deben tener validaciones activas para evitar datos vacíos o erróneos.
    Simulación de Compra: Configurar un formulario de pago integrado con una tabla de carrito.

Revisión Final (QA): * Verificar que la navegación sea fluida.
Comprobar que el diseño no se rompa en móviles (Responsividad).
Recibir feedback y aplicar ajustes finales.
